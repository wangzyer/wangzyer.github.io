内置函数内置函数

总结一些JavaScript内置函数，总结整理原则如下：

  • 常用函数，但是会忽略部分特性
  • 结合某些特出场景使用十分有效
  • 使用过程中发现某些边际情况，不注意默写特殊情况会出现BUG等问题
  • 需要深入理解的内置函数

JSON.stringify()

JSON.stringify() 方法将一个 JavaScript 对象或值转换为 JSON 字符串，如果指定了一个 replacer 函数，则可以选择性地替换值，或者指定的 replacer 是数组，则可选择性地仅包含数组指定的属性。 —— MDN

总结

  • undefined、任意的函数以及 symbol 作为对象属性值时，JSON.stringify() 对跳过（忽略）它们进行序列化
  • undefined、任意的函数以及 symbol 作为数组元素值时，JSON.stringify() 将会将它们序列化为 null
  • 转换值如果有 toJSON() 函数，该函数返回什么值，序列化结果就是什么值，并且忽略其他属性的值。
  • 布尔值、数字、字符串的包装对象在序列化过程中会自动转换成对应的原始值。
  • 如果该参数是一个函数，则在序列化过程中，被序列化的值的每个属性都会经过该函数的转换和处理；如果该参数是一个数组，则只有包含在这个数组中的属性名才会被序列化到最终的 JSON 字符串中；如果该参数为 null 或者未提供，则对象所有的属性都会被序列化。
  • 美化输出

requestAnimationFrame

window.requestAnimationFrame() 告诉浏览器——你希望执行一个动画，并且要求浏览器在下次重绘之前调用指定的回调函数更新动画。该方法需要传入一个回调函数作为参数，该回调函数会在浏览器下一次重绘之前执行 —— MDN

页面显示和FPS

页面是一帧一帧绘制出来的，当每秒绘制的帧数（FPS）达到 60 时，页面是流畅的，小于这个值时，用户会感觉到卡顿。

setTimeout，setInterval的问题

  • 定时器无法保证准确的时间执行，如果同步任务比较费事则会导致延后。
  • 间隔时间无法确定 过短浪费性能，过长页面卡顿。
  • 页面被隐藏或最小化时仍然执行

帧

游览器每一帧所做的事情：

执行时机

根据屏幕刷新率自动调整执行时机。比如若果60hz屏幕则16ms,75hz则13.3ms。requestAnimationFrame可以保证在每一帧固定时机执行一次，但是如果函数内部操作复杂会导致卡顿。

requestIdleCallback

window.requestIdleCallback()方法将在浏览器的空闲时段内调用的函数排队。这使开发者能够在主事件循环上执行后台和低优先级工作，而不会影响延迟关键事件，如动画和输入响应。 —— MDN

执行时机

如果一帧中所有要做的工作已经全部完成，还有空余时间则可以执行requestIdleCallback函数回调。

注意

  • 如果游览器空闲页面没有什么更新，那么requestIdleCallback可用时间就比较长，但是最多不会超过50ms。
  • 不要进行DOM操作。
  • 第二个参数设置超时时间、如果超过这个还没有执行则强制执行。
